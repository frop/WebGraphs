RECURSO GRAPH
Este recurso é responsável por amarzenar e recuperar grafos do sistema. Possui URI's distintas para cada operação.

::OPERAÇÕES
-Salvar um grafo
URI - Método - Parâmetros
/graph.json POST graph: JSON do grafo a ser salvo.
--Resposta
JSON com o identificador do grafo salvo.
     graphId | <id>

-Recuperar um grafo
URI - Método
/graph/<id>.json GET
--Resposta
json do grafo correspondente ao identificador <id>
    id | <id>
    vertexes | bla bla
    adjacency | ...

RECURSO ALGORTIHM
Este recurso é responsável por executar os diversos algortimos do sistema.

::OPERAÇÕES
-Obter informações sobre determinado algoritmo
URI - Método
/<alg>.json GET
--Resposta
JSON com as seguintes informações sobre o algoritmo <alg>:
	TABELA COM OS DADOS

-Executar um algortimo
URI - Método - Parâmetros
/<alg>.json POST graph: lista de identificadores de grafos de entrada. São pares do tipo nome => id, onde nome indica o que aquele grafo representa para o algoritmo.
				 param: lista de parâmetros de entrada. São pares do tipo nome => valor
--Resposta
JSON com o identificador do resultado gerado.
      result | <id>
Execuções idênticas, isto é, do mesmo algortimo e com os mesmos parâmetros, embora possam gerar resultados iguais, são armazenados sob identificadores diferentes.

RECURSO RESULT
Este recurso é responsável por recuperar informações sobre o resultado gerado por uma execução de um algoritmo.

::OPERAÇÕES
-Obter resultado
URI - Método
/result/<id>.json GET
--Resposta
JSON do resultado correspondente ao identificador <id>
	id: identificador único do resultado de uma determinada execução.
	algorithm: lista de informações sobre o algoritmo executado. Contém:
		-id: identificador do algoritmo.
		-description: breve descrição provida pelo desenvolvedor do algoritmo.
	parameter: lista de parâmetros passados para a execução.
	graph: lista com os identificadores dos grafos utilizados pelo algoritmo.
	result: lista com os resultados em si, na forma de par chave/valor.
	
RETORNO DE ERROS
Caso algum erro ocorra durante o acesso a algum dos recursos, é retornado ao cliente um json com informações sobre o erro
	'error' | 'descricao do erro'
